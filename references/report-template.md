# Markdown 报告模板

报告语言跟随用户输入；曲名、艺人、mix/version 和平台名保留官方原文。下面的中文标题只是语义槽位，实际输出时必须把标题、表头、状态标签和理由翻译成用户语言；英文请求不要照抄中文标题。

使用以下结构。四个版本是默认交付；只有用户明确要求“只给一张清单/只要综合版/单一结果”时才省略专项版。用户明确要求简版时可以压缩理由，但仍保留四个版本、验证链接和假设。

```markdown
# [Playlist Theme]

## 需求理解

- 场景：...
- 风格：...
- 情绪 / 能量：...
- BPM：... / 未验证
- 搜索平台：跨平台 / 指定平台
- 平台策略：首选 ...；允许 ...；禁用 ...；exclusive / preferred / cross-platform
- 导出意图：未请求 / 仅解释 / 准备 / 已确认执行
- 导出目标：未指定 / [用户指定平台]
- 动态内容优先顺序：场景 → 风格 → 流行度
- 数量口径：综合版目标 ... 首；三个专项版各 ... 首；候选不足时实际 ... 首
- 关键假设：...

## Digging 摘要

[2–4 句说明搜索方向、候选池规模、去重后数量、主要来源和明显限制。]

## 风格优先版（实际数量 / 目标数量）

| # | Track | Artist | 匹配 | 入选理由 | Link |

| 1 | Official Title | Official Artist | 高 | ... | [SoundCloud](direct-url) |

[同样表格]

[同样表格。说明“流行度”如何对应用户的热门或冷门意图。]

## 综合版（实际数量 / 目标数量）

## 缺失与不确定信息

- 未验证 BPM：...
- 目标平台未找到：...

## 来源

- 只列实际使用且符合平台策略的来源；禁用平台不要出现在此处。exclusive 模式下，用户报告只列目标平台直接页，其他权威来源可用于内部核验但不输出 URL。
```

## 曲目链接规则

- 链接文字使用允许的平台名，指向具体曲目页；preferred 模式按用户平台顺序排列。
- 同一曲目有多个允许的平台链接时，可以并列显示，但只占一个曲目位置。
- exclusive 模式只显示目标平台直接链接；其他平台只能在内部验证，不放进用户报告。
- 不使用搜索结果页、首页或无法确认内容的短链接。
- 来源引用应靠近其支持的事实，不用一条泛化来源替整张歌单背书。

## 长度控制

默认四个版本会产生较长报告。保持每首理由在一句以内，专项版不重复写完整元数据。综合版以播放顺序和段落为主；跨版本共性放到“跨视角核心曲目”统一解释。

如果候选少于目标数量，在标题中显示实际数量，不复制曲目凑数。
